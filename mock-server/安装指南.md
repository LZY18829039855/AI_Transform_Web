# Mock Server 安装指南

## 问题：'npm' 不是内部或外部命令

这个错误表示系统中没有安装 Node.js，或者 Node.js 没有添加到系统 PATH 环境变量中。

## 解决方案

### 方案 1：安装 Node.js（推荐）

1. **下载 Node.js**
   - 访问官网：https://nodejs.org/
   - 下载 LTS（长期支持）版本
   - 推荐版本：v18.x 或 v20.x

2. **安装 Node.js**
   - 运行下载的安装程序
   - 按照向导完成安装
   - **重要**：安装时确保勾选 "Add to PATH" 选项

3. **验证安装**
   - 关闭所有命令提示符和 PowerShell 窗口
   - 重新打开新的命令提示符或 PowerShell
   - 运行以下命令验证：
     ```cmd
     node --version
     npm --version
     ```
   - 如果显示版本号，说明安装成功

4. **启动 Mock Server**
   ```cmd
   cd E:\Huawei\ai--dashboard-main\mock-server
   npm install
   npm run dev
   ```

### 方案 2：手动配置 PATH 环境变量

如果 Node.js 已安装但 npm 命令不可用：

1. **找到 Node.js 安装目录**
   - 常见位置：
     - `C:\Program Files\nodejs`
     - `C:\Program Files (x86)\nodejs`
     - `C:\Users\你的用户名\AppData\Local\Programs\nodejs`

2. **添加到 PATH**
   - 右键"此电脑" → "属性"
   - 点击"高级系统设置"
   - 点击"环境变量"
   - 在"系统变量"中找到"Path"，点击"编辑"
   - 点击"新建"，添加 Node.js 安装目录路径
   - 点击"确定"保存
   - **重要**：关闭所有命令窗口，重新打开后再使用

3. **验证配置**
   ```cmd
   npm --version
   ```

### 方案 3：使用完整路径运行（临时方案）

如果不想修改环境变量，可以使用完整路径：

```cmd
cd E:\Huawei\ai--dashboard-main\mock-server

REM 假设 Node.js 安装在 C:\Program Files\nodejs
"C:\Program Files\nodejs\npm.cmd" install
"C:\Program Files\nodejs\npm.cmd" run dev
```

## 快速检查

运行以下命令检查 Node.js 是否已安装：

```powershell
# PowerShell
where.exe node
where.exe npm

# 或检查常见位置
Test-Path "C:\Program Files\nodejs\node.exe"
Test-Path "C:\Program Files (x86)\nodejs\node.exe"
```

## 常见问题

### Q: 安装 Node.js 后仍然提示找不到 npm？
**A:** 
1. 确保安装时勾选了 "Add to PATH"
2. 关闭所有命令窗口，重新打开
3. 检查 Node.js 安装目录是否在 PATH 中

### Q: 如何查看当前的 PATH 环境变量？
**A:** 在 PowerShell 中运行：
```powershell
$env:Path -split ";"
```

### Q: 可以使用其他 Node.js 版本管理器吗？
**A:** 可以，常用的有：
- **nvm-windows**: https://github.com/coreybutler/nvm-windows
- **fnm**: https://github.com/Schniz/fnm

## 安装完成后

1. 进入 mock-server 目录
2. 安装依赖：`npm install`
3. 启动服务：`npm run dev`
4. 服务将在 http://localhost:8080 上运行
